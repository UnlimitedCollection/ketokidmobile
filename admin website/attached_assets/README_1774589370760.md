# Admin Features & Capabilities

## 1. 📊 Admin Dashboard
Admin gets a full system overview:

- Total number of:
  - Doctors
  - Kids (patients)
  - Foods
  - Recipes
- Token status:
  - Active tokens
  - Token activity

👉 Purpose: Monitor system usage & health

---

## 2. 👤 User Management (Doctors & Kids)

### What Admin can do:
- Create new doctor accounts  
- Edit doctor details  
- Delete doctors  
- Create child (patient) profiles  
- Update child information  
- Delete incorrect/duplicate child records  

👉 Admin fully controls who can use the system

---

## 3. 🍎 Food Management

### Admin can:
- Add new foods  
- Edit food details  
- Update nutrition values:
  - Carbs
  - Fat
  - Protein
  - Calories  
- Manage food hierarchy:
  - Nutrition → Food  
- Delete outdated foods  

👉 Ensures accurate nutrition data for calculations

---

## 4. 🍽️ Recipe Management

### Admin can:
- Create recipes  
- Edit ingredients & nutrition values  
- Update recipe data  
- Delete unused recipes  

👉 Controls predefined meal options

---

## 5. 🍱 Meal Management

### Admin can:
- Create meal types:
  - Breakfast
  - Lunch
  - Dinner
  - Snack 1 / Snack 2  
- Edit meal structure  
- Delete unused meal types  

👉 Defines how the system structures daily meals

---

## 6. 🔑 Token Management (Very Important)

### Admin can:
- Generate login tokens for parents  
- Assign tokens to specific child accounts  
- Reset tokens (security issues)  
- Issue new tokens  
- Track token status:
  - Active
  - Used
  - Expired  

👉 This is the core access control system

---

## 7. ⚙️ System Control & Data Integrity

### Admin responsibilities:
- Ensure all data is accurate  
- Maintain database consistency  
- Control system-level configurations  
- Keep food & recipe data reliable  

👉 Admin ensures system correctness

---

## 8. 🛠️ System Support & Maintenance

### Admin can:
- Monitor system activity  
- Fix user issues (login/token problems)  
- Update system data  
- Maintain platform stability  

👉 Admin acts as system operator + support

---

## 🧠 Simple Summary (Easy to Understand)

👉 Admin is NOT medical  
👉 Admin is NOT parent  
👉 Admin is the system manager  

### In simple terms:

Admin controls:

- Users → Doctors & Kids  
- Data → Foods & Recipes  
- Structure → Meals  
- Access → Tokens  
- System → Stability & accuracy  

---

## 🎯 One-Line Explanation

👉 Admin manages the entire platform behind the scenes to keep everything running correctly and securely.
